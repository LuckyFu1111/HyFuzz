## How to Get Listed

If you've contributed to HyFuzz, you can be added to this file by:

1. **Code Contributions**: Submit a pull request that gets merged
2. **Documentation**: Improve documentation through pull requests
3. **Bug Reports**: Report bugs that lead to improvements
4. **Reviews**: Provide thorough reviews of pull requests
5. **Community Support**: Help others in GitHub Discussions

We aim to recognize all contributions, no matter how small!

## Acknowledgments

### Research & Development

This project was developed as part of academic research into intelligent fuzzing methodologies combining Large Language Models with traditional security testing approaches.

### Inspiration & Prior Work

HyFuzz builds upon and is inspired by:

- **AFL/AFL++**: Coverage-guided fuzzing techniques
- **AFLNet**: Protocol-aware fuzzing approaches
- **libFuzzer**: In-process fuzzing concepts
- **LLM Research**: Chain-of-Thought reasoning applications
- **IoT Security Research**: Protocol-specific vulnerability patterns

### Tools & Libraries

We gratefully acknowledge the open-source projects that make HyFuzz possible:

#### Python Libraries
- **aiohttp**: Async HTTP server/client
- **Ollama**: Local LLM inference
- **NetworkX**: Graph database operations
- **pandas/numpy**: Data processing
- **pytest**: Testing framework
- **black/isort/pylint**: Code quality tools

#### System Tools
- **strace/ltrace**: System call tracing
- **gdb**: Debugging and crash analysis
- **perf**: Performance monitoring
- **Docker**: Containerization

### Community

Thanks to the broader security research and fuzzing communities for:
- Sharing knowledge and techniques
- Providing feedback on approaches
- Contributing to open-source security tools
- Advancing the state of automated security testing
